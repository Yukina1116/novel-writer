4. AI 呼び出しのリクエストフロー（クォータチェック）

sequenceDiagram
    participant UI as React UI
    participant Auth as Firebase Auth
    participant API as /api/ai/novel/generate
    participant FS as Firestore
users/{uid}
    participant V as Vertex AI

    UI->>Auth: getIdToken()
    Auth-->>UI: JWT
    UI->>API: POST + Bearer JWT
    API->>API: verifyIdToken()
    alt Tier 0 (匿名)
        API-->>UI: 401 / 403
    else Tier 1/2
        API->>FS: get users/{uid}
        Note over API,FS: plan, usage.{yyyymm}.count を読む
        alt クォータ超過
            API-->>UI: 429 Too Many Requests
        else OK
            API->>V: generateContent()
            V-->>API: result
            API->>FS: increment usage.{yyyymm}.count
            API-->>UI: 200 + result
        end
    end

5. 暗号化バックアップ（opt-in、有料会員のみ）

sequenceDiagram
    participant UI as React UI
    participant CR as Web Crypto API
    participant API as /api/backup/upload
    participant GCS as Cloud Storage
backups/{uid}/

    Note over UI,CR: 初回: ユーザーがパスフレーズ設定
(サーバーに送らない)
    UI->>CR: PBKDF2(passphrase, salt) → key
    UI->>UI: IndexedDB から全データ読み出し
    UI->>CR: AES-GCM encrypt(JSON, key)
    CR-->>UI: ciphertext + iv

    UI->>API: 署名URL リクエスト
    API->>API: plan === 'paid' ? 確認
    API->>GCS: 署名 PUT URL 発行
    GCS-->>API: signed URL
    API-->>UI: signed URL

    UI->>GCS: PUT ciphertext (直接アップロード)
    GCS-->>UI: 200

    Note over UI,GCS: サーバーは平文を一切見ない
パスフレーズ紛失=復号不能

6. 移行ロードマップ

順PR内容規模
1Firebase Auth + AI GateGoogle ログイン、`/api/ai/*` 全保護、匿名は 401中
2レートリミット + 月次クォータusers/{uid}.usage.{yyyymm} で増分管理、429 返却中
3IndexedDB (Dexie) 移行 + Export/ImportFirestore projects → IndexedDB 一括移送、バックアップ警告UI大
4Firestore スキーマ整理projects コレクション削除、users/{uid} のみ小
5Stripe SubscriptionCheckout + Webhook、plan 状態機械実装中
